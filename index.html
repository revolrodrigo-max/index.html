Documentación del Dashboardv3.1
Arquitectura & Flujo de datos
1
Google Sheets (Fuente) — Planilla testo_boost_base_datos_v2 con 7 hojas: ÍNDICE, EQ1_CIERRES, EQ2_CIERRES, EQ3_CIERRES, ADHERENCIA, DISPONIBILIDAD, PROFESIONALES. Los datos se cargan manualmente por el equipo.
2
Publicación CSV — Cada hoja está publicada como CSV público via File → Share → Publish to Web. URLs con formato pub?gid=XXX&output=csv.
3
Dashboard HTML — Single-file app (index.html) alojada en GitHub Pages. Consume los CSVs via fetch(), parsea y renderiza. Sin backend ni base de datos.
4
Actualización — Click en "Actualizar" o al cargar la página. Los datos se refrescan en tiempo real desde Google Sheets (cache-bust con timestamp).

Notas técnicas
Formato de fechas: DD/MM/AAAA — parseado con split('/'). La semana se calcula lunes a domingo (ISO).
Cache-bust: Cada fetch agrega &cachebust=Date.now() para evitar cache del navegador y mostrar datos en tiempo real.
Sin backend: 100% client-side. No hay base de datos ni servidor. Los datos viven en Google Sheets y se leen como CSV publicado.
Hosting: GitHub Pages — repositorio revolrodrigo-max.github.io. Deploy automático al hacer push a main.
